This website features the ability for a prospective graduate student to apply for this grad school.
The application gets reviewed by faculty before being finally approved. This application was integrated with
two other applications to create a full-fledged portal for graduate students. Thus, this website required
tons of backend development as well as front end development. The backend was constructed with Flask.
The database was created with SQL. This was all done for my Databases Design Class. What you see here was
created with 2 other people over the course of a couple of weeks.

To delve in to the site from a more technical point, it allows a user to fill out an application - this application includes
uploading files (like transcript), emailing the "university", and even includes an automated mailer.
All the data in the application gets appropriately parsed and stored into the SQL DB. Along with filling out
an application, users have to make an account to go along with it. There are also faculty accounts with different roles.
These roles include judging applications, accepting transcripts, viewing reccomendation letters, and accepting applicants.
There is a centralized faculty page for reviewing applications and depending on the certain role, thanks to Flask, the appropriate features
get shown on to the page. The creation of this application
After integrating the site with another webpage that allows users to register for classes, get grades, etc. I had to add a feature to matriculate applicants from applicant to student.
Also created a "main" portal page for faculty and students which shows the approrpiate content based on the "role".
